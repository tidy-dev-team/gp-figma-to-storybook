## Component-Specific Constraints

_(Override per component as needed.)_

- All interactive elements must use KendoReact primitives — never raw HTML `<input>`, `<select>`, or `<button>`.
- All `RadioButton` groups must share a `name` prop so the browser enforces single-select.
- Layout via CSS flexbox/gap — no absolute positioning.
- Every prop that feeds into `useState` must have a matching `useEffect` to sync it (enables live Storybook Controls).
- No Storybook imports inside component files (`src/components/**`).

---

## Story Conventions

- File extension: `.stories.tsx` — never `.ts` (Vite rejects JSX in non-TSX files).
- Format: CSF3 with `satisfies Meta<typeof X>` and `StoryObj<typeof meta>`.
- Every prop: `argTypes` entry with `control`, `description`, and `options` (for enums).
- Aim for 4–8 named story variants per component, plus an `AllVariants` story.
- Use realistic text from the Figma design as initial `args` — no "foo" / "bar" placeholders.
- `AllVariants` story renders all meaningful states side-by-side in a column layout.

---

## Accessibility Target

- WCAG AA minimum.
- Minimum touch target: 44×44 px where applicable.
- Focus ring: visible on all interactive elements; use `accent-color` (`#00aecf`).

# Genpact DS Quality Standards

> Acceptance criteria for every component generated by `/genpact-figma-to-storybook`. Items marked ⚙️ are checked by TypeScript / linting; items marked 👁️ require visual sign-off in Storybook.

---

## ❗ Two-File Rule — STRICT

**Every component lives in two files.** No exceptions.

- `src/components/<Name>/<Name>.tsx` — the React component
- `src/stories/<Name>.stories.tsx` — the Storybook story

The component file must never import from `@storybook/*`. The story file must never define component logic — only import and configure.

---

## 🏗️ File & Export Structure

- Component file exports: the component function, its props type, and any enums needed by the story.
- Story file exports: `default` (meta) + one named export per story variant.
- Props type is named `<ComponentName>Props` and exported.
- No barrel `index.ts` files — import directly from the component file path.

---

## 🎨 Design Tokens

- All colors come from `DESIGN.md` tokens, defined in a `const t = { ... }` block at the top of the component file.
- No hardcoded hex values outside `const t`.
- Token comments reference the DESIGN.md name: `// token: accent-color`.
- Spacing uses 4px-grid multiples unless a specific Figma value is documented.

---

## ⚙️ TypeScript

- `npx tsc --noEmit` must pass with zero errors before a component is considered done.
- Props types use `string`, `string[]`, `Date | null` etc. — no `any`.
- Enum-style props use a `type` alias exported from the component file (e.g., `export type FilterMode = 'savings' | 'benchmarking' | 'report'`).

---

## 🔄 Storybook Controls

- Every prop that maps to `useState` has a `useEffect` sync:
  ```tsx
  useEffect(() => { setState(prop); }, [prop]);
  ```
- Changing a Control in Storybook must update the rendered component live — verify by toggling Controls in the browser before marking done.
- `argTypes` `control` types match the prop: `'radio'` for small enums, `'select'` for longer lists, `'object'` for arrays, `'date'` for `Date`, `'boolean'` for booleans.

---

## 🧩 KendoReact Primitives

- Form elements always use KendoReact: `RadioButton`, `Checkbox`, `Input`, `DropDownList`, `MultiSelect`, `DatePicker`, `DateRangePicker`, `Button`, etc.
- Never use `<input>`, `<select>`, `<textarea>`, or `<button>` directly.
- If a visual element has no KendoReact equivalent (e.g., a custom progress bar, a step indicator), implement it as pure CSS/SVG with semantic HTML and flag it in the report.
- All KendoReact components inherit the Default theme from `.storybook/preview.ts` — do not import theme CSS in component files.

---

## 📐 Layout

- CSS flexbox or grid for all layout — no `position: absolute` (except decorative overlays or badges).
- All spacing (padding, gap, margin) expressed as pixel numbers in inline style objects — no magic numbers outside `const t`.
- Component width: `inline-flex` or `inline-grid` by default so it hugs content; use `width: '100%'` only when the Figma design fills the container.

---

## 🔣 Icons & SVG

- Inline SVG is acceptable for simple custom icons (e.g., checkmark in a step bullet).
- Icon paths use `currentColor` or a token color — never raw hex directly on the `<path>`.
- All SVGs include `viewBox` and `fill="none"` where strokes are used.

---

## 👁️ Visual Sign-Off Checklist

Before marking a component done, confirm all of the following in the Storybook browser:

| # | Check | How |
|---|---|---|
| 1 | `npx tsc --noEmit` passes with zero errors | ⚙️ terminal |
| 2 | All story variants render without console errors | ⚙️ browser console |
| 3 | `AllVariants` story matches all Figma states visually | 👁️ side-by-side with Figma screenshot |
| 4 | Changing Controls updates the component live | 👁️ toggle Controls panel |
| 5 | Colors match Genpact brand tokens exactly | 👁️ eyedrop against Figma |
| 6 | Font is Rubik, sizes match Figma | 👁️ inspect element |
| 7 | No raw HTML form elements (`<input>`, `<select>`, `<button>`) | ⚙️ grep / code review |
| 8 | Layout survives viewport resize (no overflow, no broken grid) | 👁️ resize browser window |
| 9 | Both files exist; component file has no Storybook imports | ⚙️ code review |
| 10 | `useEffect` sync exists for every stateful prop | ⚙️ code review |
