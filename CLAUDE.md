# Genpact Integration — Claude Code Context

This project scaffolds Storybook stories for Genpact's Design System components, built from KendoReact primitives.

## Project Structure

- `src/components/` — React component files (one folder per component)
- `src/stories/` — Storybook story files (`.stories.tsx`, one per component)
- `KENDOREACT_COMPONENTS.md` — Reference: Figma visual patterns → KendoReact component API
- `DESIGN.md` — Genpact brand tokens (generated by `/ds-extract-design`, run once)
- `REQUIREMENTS.md` — Design standards and quality acceptance criteria for this project
- `skills/genpact-figma-to-storybook.md` — The story-generation skill definition
- `.storybook/preview.ts` — Imports KendoReact Default theme globally

## Available Skills

### `/genpact-figma-to-storybook <figma-url>`

Reads a Figma component from the Genpact Design System and generates a production-ready Storybook CSF3 story file using KendoReact sub-components.

**Full skill definition:** `skills/genpact-figma-to-storybook.md`

**Quick reference:**
1. Parse fileKey + nodeId from URL
2. Call `get_design_context` via Figma MCP
3. Decompose visual elements → map to KendoReact via `KENDOREACT_COMPONENTS.md`
4. Identify Figma variants → PascalCase story names
5. Generate component wrapper + story file at `src/stories/<ComponentName>.stories.tsx`

## Conventions

- Story files: `.stories.tsx` (never `.ts` — Vite requires TSX extension for JSX)
- Story format: CSF3 with `satisfies Meta<typeof X>` and `StoryObj<typeof meta>`
- All interactive elements must use KendoReact components (never raw HTML inputs)
- Brand tokens: `#15223F` text, `#104683` primary blue, `#C1CBE1` border, `#E3E7EF` filter bg
- Font: Rubik (Light 300, Regular 400), base 12px

## Running Storybook

```bash
npm install
npm run storybook
```

Opens at `http://localhost:6006`
